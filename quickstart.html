QUICKSTART

Your first matter, judgment to freeze packet

Mint a sandbox key, open a matter, add parties, attach a judgment, trace a wallet, build an enforcement packet, and verify the audit chain. Every step is a copy-paste curl command against the live sandbox API.

API version 2026-07-05 45 ops / 11 resources Base URL api.kineticanvil.com
KEY
Mint a sandbox key

Issue a scoped sandbox key from the site passthrough. Shown once; copy it before you close the tab.

curl 
curl -s -X POST https://kineticanvil.com/api/sandbox-key \
  -H "Content-Type: application/json" \
  -H "User-Agent: curl/8.5.0" \
  -d '{}'
201 Response 
{
  "key": "kx_test_...",
  "key_id": "key_b019d27c0687",
  "environment": "sandbox",
  "scope": "sandbox:read sandbox:write",
  "base_url": "https://api.kineticanvil.com",
  "rate_limit": "5000 req/min per key",
  "note": "Shown once. This key is signed, not stored - it cannot be recovered. Lost it? Generate a new one.",
  "docs": "https://docs.kineticanvil.com"
}

Save the key as ANVIL_KEY. The scope label is a display string; the key carries all ~55 non-privileged sandbox grants.

01
Open a matter

Create the case record. Everything else anchors to the matter_id returned here.

curl · POST /v1/matters 
curl -s -X POST https://api.kineticanvil.com/v1/matters \
  -H "Authorization: Bearer $ANVIL_KEY" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: gp-matter-$(date +%s)" \
  -d '{
    "name": "Varela v. Zhao - BTC Recovery",
    "mode": "recovery",
    "lead_counsel": "Sarah L. Varela, Esq."
  }'
201 Response 
{
  "name": "Varela v. Zhao - BTC Recovery",
  "mode": "recovery",
  "lead_counsel": "Sarah L. Varela, Esq.",
  "matter_id": "matter_3ee6c9d0418fz52bdq",
  "id": "matter_3ee6c9d0418fz52bdq",
  "organization_id": "org_8eefaa57986d",
  "status": "active",
  "created_at": "2026-07-05T20:33:57.005Z"
}

Note the matter_id. Substitute it wherever you see matter_3ee6c9d0418fz52bdq in the steps below.

03
Attach the judgment

Record the underlying judgment. All seven required fields must be present at intake; they cannot be added by PATCH later.

curl · POST /v1/judgments/intake 
curl -s -X POST https://api.kineticanvil.com/v1/judgments/intake \
  -H "Authorization: Bearer $ANVIL_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "matter_id": "matter_3ee6c9d0418fz52bdq",
    "amount": 285000,
    "court": "Southern District of New York",
    "case_number": "1:24-cv-04471",
    "date_entered": "2024-11-15",
    "jurisdiction": "United States - SDNY",
    "creditor": "Elena Varela",
    "debtor": "Wei Zhao"
  }'
201 Response 
{
  "matter_id": "matter_3ee6c9d0418fz52bdq",
  "court": "Southern District of New York",
  "case_number": "1:24-cv-04471",
  "jurisdiction": "United States - SDNY",
  "creditor": "Elena Varela",
  "debtor": "Wei Zhao",
  "amount": 285000,
  "date_entered": "2024-11-15",
  "judgment_id": "judgment_7c716919f8eftbuwa5",
  "id": "judgment_7c716919f8eftbuwa5",
  "object": "judgment",
  "status": "active",
  "created_at": "2026-07-05T20:34:21.005Z"
}

Required at intake: matter_id, amount, court, case_number, date_entered, jurisdiction, creditor, debtor. A missing field returns a 422 listing exactly what is absent.

Set enforcement posture

curl · POST /v1/judgments/{id}/enforcement

  https://api.kineticanvil.com/v1/judgments/judgment_7c716919f8eftbuwa5/enforcement \
  -H "Authorization: Bearer $ANVIL_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "strategy": "crypto_trace_and_freeze",
    "venues": ["binance","okx"],
    "priorities": ["trace","freeze","disclose"]
  }'

{

  "review_required": true,

06
Dispatch crypto trace

Submit a wallet cluster for asynchronous tracing. Returns 202 immediately; a background worker runs the multi-hop flow map.

curl · POST /v1/crypto/asset-map 
curl -s -X POST https://api.kineticanvil.com/v1/crypto/asset-map \
  -H "Authorization: Bearer $ANVIL_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "matter_id": "matter_3ee6c9d0418fz52bdq",
    "wallets": ["1A1zP1eP5QGefi2DMPTfTL5SLmv7Divfna"],
    "chains": ["bitcoin"],
    "include_exchanges": true
  }'
202 Response (dispatched) 
{
  "object": "crypto_asset_map",
  "map_id": "amap_21cb17f7cf53pu1u73",
  "wallets": [],
  "token_exposures": [],
  "exchange_hypotheses": [],
  "stablecoin_exposure": [],
  "status": "processing",
  "job_id": "job_21cb17f7cf531bvizm"
}

Note the job_id for the next step. The 202 confirms the job is queued; status is not yet complete.

Build an enforcement packet

Assemble the judgment, recovery event, and wallet set into a signed enforcement artifact. Packet type must be exchange_freeze for a VASP freeze request.

curl · POST /v1/packets 
curl -s -X POST https://api.kineticanvil.com/v1/packets \
  -H "Authorization: Bearer $ANVIL_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "matter_id": "matter_3ee6c9d0418fz52bdq",
    "type": "exchange_freeze",
    "judgment_id": "judgment_7c716919f8eftbuwa5",
    "recovery_event_ids": ["rcv_e4007d3a1ad6iz4rkg"],
    "wallets": ["1A1zP1eP5QGefi2DMPTfTL5SLmv7Divfna"],
    "target_venue": "binance"

11
Read the matter dashboard

Pull the live phase snapshot. Shows recovery value, open packets, and the current phase on the enforcement state machine.

curl · GET /v1/matters/{id}/dashboard 
curl -s \
  https://api.kineticanvil.com/v1/matters/matter_3ee6c9d0418fz52bdq/dashboard \
  -H "Authorization: Bearer $ANVIL_KEY"
200 Response 
{
  "matter_id": "matter_3ee6c9d0418fz52bdq",
  "judgment_value": { "amount": 128000, "currency": "USD" },
  "known_crypto_exposure": { "amount": 128000, "currency": "USD" },
  "open_actions": 1,
  "open_packets": 1,
  "alerts": 0,
  "recovery_events": 1,
  "recovery_phase": "dispatched",
  "phase_index": 3,
  "remittances": 0,
  "packets": 1
}

Phase machine: intake → traced → packaged → dispatched → remitting → settled → closed. Phase dispatched (index 3) is expected here.

12
Verify the audit chain

Confirm the tamper-evident event log is unbroken. Returns the chain head hash and a count of verified events.

curl · GET /v1/events/verify-chain 
curl -s https://api.kineticanvil.com/v1/events/verify-chain \
  -H "Authorization: Bearer $ANVIL_KEY"
200 Response 
{
  "object": "chain_verification",
  "valid": true,
  "events_checked": 17,
  "first_break": null,
  "head": "ed246d2ef5eb2c8c5b688f054bccb3545bb82e28adf55a965dba252b49ab0d44",
  "verified_at": "2026-07-05T20:37:35.406Z"
}

Chain valid. 17 events. No breaks. The same endpoint can be called at any point in a matter lifecycle to confirm integrity.
